八戒客php-zzgg-un-web项目环境搭建详解

❅以下php环境源于Docker搭建，以下步骤请至上而下依次操作执行，同时该文档暂只支持Mac下环境搭建,同时，本篇文章仅适用于猪八戒公司下八戒客项目的搭建。

一、前端环境搭建

​ 1.获取前端项目gitlab权限

​ [fe/rake-zbj]、[zbj-fe/fis-common]、[fe/fis-agent]、[fe/fis-mobile] 

2.将所有的项目放置在同一级目录下

​ 3.搭建rk环境 https://git.zhubajie.la/fe/rake-zbj （java只能装JDK8版本，否则项目释放不成功）

​ 4.分别到[zbj-fe/fis-common]、[fe/fis-agent]、[fe/fis-mobile]下释放资源

​ 5.获取server.conf服务配置 （在fis-agent下执行）

​ 6.初始化服务 （在fis-agent下执行）

​ 7.开始服务 （在fis-agent下执行）

​ 8.保持释放与监听 （在fis-agent下执行）

二、PHP环境的搭建

  • 准备工作

    1、安装好Docker、Composer、Git等必要工作

    Docker：https://www.docker.com/

    Composer：https://pkg.phpcomposer.com/

    2、获取项目gitlab权限dept-ad-zbjagent /php-zzgg-un-web

    3、新建文件夹rake-static、rake-zbj与php-zzgg-un-web同级 ，同时将SlightPHP、receiver.php放置到同级（五个文件都在同级）

  • 获取php-zzgg-un-web镜像

  • 获取SlightPHP资源

    怎么获取？？(暂时找谢翔)

  • 在Docker中引入镜像

    查看镜像是否已引入

  • 创建容器

    注意：/Users/xuxingxing/Projects/bajieke：设置为本地项目放置路径（包含有rake-static、rake-zbj、SlightPHP、receiver.php、php-zzgg-un-web的目录名称，其他不做修改，80端口如果被占用可以使用其他端口），启动后的容器名称为php-zzgg-un，容器里面的映射目录为 /var/www/html。

  • 启动容器

    docker ps命令查看已启动容器，里面如果没有启动所创建的容器或者关机重启电脑之后，需要执行以下命令启动容器即可

  • 绑定本机hosts

    192.168.143.252 api.config.zhubajie.la #api的配置中心
    127.0.0.1 un.dev.zbjdev.com #项目本地访问地址
    127.0.0.1 receiver.xxx.com #前后端传输接收文件 xxx需要改为名字首拼
    127.0.0.1 a.dev.zbjdev.com #配置接收文件的hosts

  • 修改容器里面的nginx对应的host配置

修改server_name receiver.xx.com 为 server_name receiver.名字首拼.com （与上一条绑定hosts相对应）

  • 配置系统的接入

    接入说明：https://git.zhubajie.la/zbj_base/config-client/wikis/how-to （可以忽略不看，已在容器里面配置好）

    1.在本地项目php-zzgg-un-web里面执行

    2.到容器里面更新配置

    3.重启容器

    这个时候能顺利访问 receiver.名字首拼.com/receiver.php （eg:receiver.xx.com/receiver.php） 显示出来： I’m ready for that, you know.

❅ OK!这个时候就完成95%的工作量啦，现在我们要做的就是要将前端的模板推送到后端，让后端来编译前端文件。

  • 前端模板推送到后端

    在本地三个前端项目里面都需要添加推送配置

    zbj-fe/fis-common、fe/fis-agent、fe/fis-mobile项目的根目录下分别有fis-conf.js文件

    配置里面的所有xxx都需要设置为名字首拼，与绑定的hosts一致，其他无需做更改。

    • 执行前端模板推送到后端

      分别到本地三个前端项目中执行 rk release -Dd remote_xxx （xxx为配置的名字首拼，与fis-config.js相对应）

      前端代码的更改都需要执行解析推送到后端，否则更改代码后页面显示无效，如需实时监听更改，请在fis-agent执行

      恭喜！此刻就能顺利访问http://un.dev.zbjdev.com看到成果啦！
